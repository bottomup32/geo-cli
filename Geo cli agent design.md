## 1. 프로젝트 개요

### 1.1 GEO란?

**GEO (Generative Engine Optimization)** 는 AI 시대의 SEO다.

브랜드, 제품, 서비스, 또는 특정 주제에 대해 **AI가 어떻게 알고 있고 어떤 답을 주는지** 정량적으로 평가하고, 그 분석 결과에 따라 원하는 방향으로 AI 응답을 개선하는 전략을 수립하는 과정이다.

기존 SEO가 Google 검색 결과에서의 노출을 최적화했다면, GEO는 ChatGPT, Google AI Mode 같은 생성형 AI 응답에서의 노출과 포지셔닝을 최적화한다.

### 1.2 해결하는 문제

- 내 브랜드/제품을 AI가 어떻게 묘사하는지 모른다
- 경쟁사 대비 AI 응답에서 얼마나 언급되는지 알 수 없다
- AI 응답에 영향을 주는 소스(URL)가 무엇인지 파악이 안 된다
- 콘텐츠를 개선했을 때 AI 응답이 실제로 바뀌는지 추적하기 어렵다
- 이 모든 것을 비기술 비즈니스 사용자가 쉽게 실행할 수 없다

### 1.3 사용 대상

- **주 사용자:** 비기술 비즈니스 사용자 (마케터, 브랜드 매니저, 경영진)
- **활용 맥락:** TecAce 컨설팅 서비스의 핵심 도구
- **목표:** 클라이언트의 브랜드/제품 GEO 분석 보고서 생성

## 2. 핵심 측정 지표

| 지표 | 설명 |
|------|------|
| **Brand Mention Rate** | 특정 쿼리에서 AI가 브랜드를 언급하는 빈도 |
| **Positioning (Rank)** | AI 응답 내 브랜드가 몇 번째로 언급되는지 순위 |
| **SoV (Share of Voice)** | 전체 언급 중 우리 브랜드가 차지하는 비중 |
| **Total Mentioned** | 전체 쿼리 중 총 언급 횟수 |
| **Sentiment / Tone** | AI가 브랜드를 긍정적/부정적/중립적으로 묘사하는 비율 |
| **Competitor Comparison** | 경쟁 브랜드 대비 상대적 위치 |
| **Reference URL** | AI가 응답에서 참조한 URL 소스 (AI 학습 소스 파악) |
| **Persona-level 분석** | 쿼리 유형별 페르소나 분류 후 지표 세분화 |

## 4. 시스템 아키텍처

### 4.1 전체 파이프라인 (MVP)

```
[사용자 입력]
     │
     ▼
┌─────────────────────────────┐
│  1. Orchestrator Agent      │  ← 인터뷰로 의도 파악
│     (대화 + 확인 폼 하이브리드) │
└─────────────┬───────────────┘
              │ 브랜드, 목적, 페르소나, 경쟁사 정의
              ▼
┌─────────────────────────────┐
│  2. Query Agent             │  ← 50~100개 자연스러운 질문 생성
│     (타겟 고객 관점 쿼리)     │
└─────────────┬───────────────┘
              │ 쿼리 목록 확인 (사용자 피드백)
              ▼
┌─────────────────────────────┐
│  3. AI Testing Agent        │  ← 비동기/병렬 AI 질의
│     (ChatGPT, Google AI)    │
└─────────────┬───────────────┘
              │ 원데이터 확인 후 분석 진행 여부
              ▼
┌─────────────────────────────┐
│  4. Analysis Agent          │  ← 정량 분석 + Persona 분류
│     → CSV/JSON 저장         │
└─────────────┬───────────────┘
              │ 분석 결과 데이터
              ▼
┌─────────────────────────────┐
│  5. Report Agent            │  ← geo-report 스킬 호출
│     (Claude 기반 컨설팅 보고서) │
└─────────────┬───────────────┘
              │ (Phase 2)
              ▼
┌─────────────────────────────┐
│  6. Dashboard Agent         │  ← 클라우드 배포 웹 앱
│     (시계열 추적 + 시각화)    │
└─────────────────────────────┘
```

### 4.2 사용자 피드백 포인트 (Human-in-the-Loop)

파이프라인 중 4개의 핵심 피드백 포인트가 존재한다:

1. **인터뷰 단계:** Orchestrator가 브랜드, 경쟁사, 페르소나, 분석 목적을 대화로 수집하고 확인 폼으로 요약 확인
2. **쿼리 생성 후:** 생성된 50~100개 쿼리 목록을 사용자가 검토/수정 후 승인
3. **데이터 수집 후:** 원데이터(AI 응답 raw data) 확인 후 분석 진행 여부 결정
4. **전략 제안 전:** 최종 전략 보고서 방향 확인 (컨퍼마코리우스)

> 분석 도중 추가 쿼리나 범위 수정이 필요한 경우에도 인터럽트 가능.

## 5. 각 에이전트 상세 설계

### 5.1 Orchestrator Agent

**역할:** 사용자의 분석 의도를 파악하고 하위 에이전트들에게 작업 지시

**수집 정보:**
- 분석 대상 (브랜드명 / 제품명 / 서비스명 / 주제)
- 분석 목적 (브랜드 인지도 확인 / 경쟁 분석 / 콘텐츠 전략 / 위기 모니터링)
- 타겟 고객 페르소나 (사용자 사전 정의 + AI 추가 유추 하이브리드)
- 경쟁사 목록
- 분석 대상 AI 플랫폼 선택
- 보고서 언어 및 독자 수준

**인터뷰 방식:** 대화형 CLI 질의 → 수집된 정보 요약 확인 폼 → 사용자 승인

**출력:** 구조화된 분석 브리프 (JSON)

---

### 5.2 Query Agent (질문 에이전트)

**역할:** 타겟 고객이 실제로 할 법한 자연스러운 질문 생성

**생성 기준:**
- 타겟 페르소나 관점에서 자연스러운 질문
- 인지적(informational), 탐색적(exploratory), 비교적(comparative) 쿼리 유형 혼합
- 다국어 동시 지원 (타겟 시장에 맞게)

**수량:** 50~100개 (1회 실행, 반복 없음)

**출력:** 쿼리 목록 (CSV / JSON) + 사용자 확인 단계

---

### 5.3 AI Testing Agent

**역할:** 생성된 쿼리를 타겟 AI 플랫폼에 질의하고 응답 수집

**수집 방법:**
- ChatGPT.com → Playwright 웹 자동화 (스크래핑)
- Google AI Mode → Playwright 웹 자동화 (스크래핑)
- API가 있는 플랫폼 → 공식 API 사용

**실행 방식:**
- 비동기 백그라운드 실행 (완료 시 알림)
- 쿼리 병렬 처리로 속도 최적화

**에러 처리:**
- 실패한 쿼리는 자동 재시도 (최대 N회)
- 재시도 후에도 실패 시 N/A 표기하고 나머지 데이터로 분석 진행

**저장:** 원시 응답 데이터 → CSV/JSON 로컬 파일

---

### 5.4 Analysis Agent (분석 에이전트)

**역할:** AI 응답을 기반으로 정량 지표 계산 및 구조화

**분석 항목:**

| 분석 항목 | 내용 |
|-----------|------|
| Visibility | 전체 쿼리 중 브랜드 언급 비율 |
| Rank | 응답 내 언급 순서 (1st mention, 2nd mention...) |
| SoV | 경쟁사 포함 전체 언급 중 우리 브랜드 비중 |
| Total Mention | 절대 언급 횟수 |
| Sentiment | 긍정/부정/중립 분류 및 비율 |
| Persona 분류 | 쿼리 유형별 페르소나 매핑 (사용자 정의 + AI 유추 하이브리드) |
| Reference URL | AI 응답에서 참조된 URL 수집 및 도메인 분류 |
| 강약점 지도 | 어떤 쿼리 유형/페르소나에서 강하고 약한지 매핑 |

**저장 포맷:** CSV + JSON (로컬 파일)

**출력:** 분석 결과 구조화 데이터 → Report Agent로 전달

---

### 5.5 Report Agent (geo-report 스킬)

**역할:** 분석 데이터를 기반으로 컨설팅 페이퍼 수준의 보고서 생성

**현황:** Claude 기반 `geo-report` 스킬로 이미 구현됨

**입력:** Analysis Agent의 JSON/CSV 결과 데이터

**출력:** 컨설팅 스타일 보고서 (한국어/다국어)

**보고서 내용 (예상 구조):**
- Executive Summary (현황 진단)
- AI 가시성 분석 (플랫폼별 Visibility, SoV, Rank)
- 감성 분석 결과
- 경쟁사 비교 분석
- 참조 소스(URL) 분석 및 AI 학습 소스 인사이트
- 페르소나별 분석 결과
- 종합 GEO 전략 제안:
  - 콘텐츠 작성/수정 가이드
  - 특정 웹페이지/사이트 최적화 권고
  - AI가 인용할 만한 보도자료/PR 소재 제안
  - 스키마 마크업 개선안

---

### 5.6 Dashboard Agent (Phase 2)

**역할:** 분석 결과를 웹 대시보드로 시각화

**주요 기능:**
- 현재 GEO 점수 시각화 (플랫폼별, 지표별)
- 시계열 추적: 콘텐츠 개선 후 AI 응답 변화 추이
- 주기적 SoV 워치 (정기 스케줄 실행)
- 경쟁사 비교 차트

**배포:** 클라우드 웹 앱 (배포 환경 미정)

## 6. 기술 스택

### 6.1 언어 및 프레임워크

| 레이어 | 기술 |
|--------|------|
| AI 에이전트 / 분석 로직 | Python |
| CLI 인터페이스 | Python (or Node.js CLI) |
| 웹 스크래핑 | Playwright (Python) |
| 웹 대시보드 | Node.js / TypeScript (Phase 2) |
| AI 모델 | Claude API (Anthropic SDK) |

### 6.2 데이터 저장

| 용도 | 저장 방식 |
|------|-----------|
| 원시 AI 응답 데이터 | CSV + 로컬 JSON 파일 |
| 분석 결과 | CSV + JSON |
| 시계열 이력 | CSV 누적 (나중에 DB 전환 가능) |
| 프로젝트 설정 | JSON 설정 파일 |

### 6.3 멀티 에이전트 조율

- Python 기반 Orchestrator가 하위 에이전트들을 순차/병렬로 호출
- 에이전트 간 통신: 구조화된 JSON 데이터 파일 또는 메모리 내 객체

## 7. 핵심 리스크 및 트레이드오프

### 7.1 웹 스크래핑 불안정성 (High Risk)

| 항목 | 내용 |
|------|------|
| **리스크** | ChatGPT.com, Google AI Mode UI 변경 시 스크래핑 코드 즉시 중단 |
| **법적 리스크** | ToS 위반 가능성 인지, 콘설팅 내부 도구로서 실용성 우선 감수 |
| **완화 방법** | 스크래핑 레이어를 별도 모듈로 분리해 업데이트 용이하게 설계 |

### 7.2 API 비용 (Medium Risk)

| 항목 | 내용 |
|------|------|
| **리스크** | 100개 쿼리 × N개 플랫폼 = 분석 1회당 상당한 API 비용 |
| **완화 방법** | 쿼리 수 조절 옵션 제공, 저비용 모드(쿼리 50개) 옵션 추가 |

### 7.3 AI 응답 비결정성

| 항목 | 내용 |
|------|------|
| **리스크** | 동일 쿼리에도 AI가 매번 다른 응답 → 통계적 노이즈 |
| **현재 결정** | 1회 실행만 하므로 스냅샷 분석으로 접근, 시계열 반복으로 추세 파악 |

## 8. 언어 지원

- **다국어 동시 지원**: 타겟 시장에 맞게 쿼리 생성 언어 설정 가능
- 기본: 영어 + 한국어
- 보고서: 클라이언트 언어에 맞게 생성

---

## 9. 차별화 포인트

기존 GEO 도구들(Semrush, BrightEdge 등)과의 차별점:

1. **자유도와 커스터마이징**: CLI 기반의 자유로운 요청 입력, 클라이언트별 맞춤 분석
2. **정확하고 필요한 쿼리 생성**: AI가 타겟 고객 관점에서 가장 관련성 높은 쿼리를 자동 생성
3. **쉬운 사용**: 비기술 비즈니스 사용자도 대화형 인터뷰로 실행 가능
4. **컨설팅 수준 보고서**: 수치 데이터를 넘어 실행 가능한 전략까지 제공
5. **특정 AI 플랫폼 특화**: ChatGPT, Google AI Mode에 최적화된 분석
