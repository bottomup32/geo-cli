# GEO CLI 로컬 실행 가이드

생성 날짜: 2026-03-26

## 개요
GEO CLI는 **로컬-퍼스트(Local-First)** 접근 방식을 사용합니다. 모든 데이터 수집, 테스트(ChatGPT 스크래핑), 분석 처리 및 파이프라인 관리는 전적으로 로컬 PC 환경에서 이루어집니다. 

외부(예: 모바일 환경 등)에서 접근이 필요한 경우, `Ngrok`을 사용하여 터널링하는 방식으로 호스팅합니다.

---

## 🚀 실행 방법 (Local)

스트림릿(Streamlit) 웹 애플리케이션을 단독으로 실행합니다. 이 방식은 PC 내부에서 직접 접근하여 사용할 때 적합합니다.

```powershell
# 최초 1회 환경 설정 및 의존성 설치 후 바로 실행
.\setup_and_run.ps1

# 이미 설치가 되어 있는 경우 스킵하고 실행
.\setup_and_run.ps1 -SkipInstall
```
접속 URL: `http://localhost:8501`

---

## 🌐 공용 접속 모드 (외부에서 스마트폰 등으로 접속할 때)

`run_public.ps1` 스크립트(추가 예정 또는 수동 터널링)를 사용하여 외부 클라이언트가 로컬 8501 포트에 접근할 수 있도록 일시적인 URL을 생성합니다. 

```powershell
# 1. 터미널 1번에서 로컬 앱 실행
.\setup_and_run.ps1 -SkipInstall

# 2. 터미널 2번에서 ngrok 실행 (포트 8501)
ngrok http 8501
```

제공된 `https://xxxx.ngrok-free.app` 주소를 통해 모바일에서 접속할 수 있습니다. 

> **참고:** 플레이라이트(Playwright) 브라우저는 서버 역할을 수행하는 로컬 PC의 화면에 표시되며, 모바일 화면에서는 조작할 수 없습니다. 따라서 테스트 진행 시점에는 PC 근처에 있거나, 로그인 세션이 이미 저장된 상태여야 합니다.

## 🔧 사전 준비물
1. Python 3.11 이상
2. `ANTHROPIC_API_KEY` 발급 후 앱 내 [⚙️ 설정] 또는 `.env` 파일에 기입

## ❗ 트러블슈팅
| 증상 | 해결 |
|------|------|
| Playwright 오류 | `playwright install chromium` 명령어 직접 실행 |
| ChatGPT 로그인 실패 | 브라우저 창에서 수동 로그인 후 5분 이내 완료 |
| 타임아웃 오류 발생 | 터널링 연결 상태, 혹은 API 키 설정 재확인 |
| ChatGPT UI 변경 | `geo_cli/agents/testing_agent.py` 상단 셀렉터 수정 |
