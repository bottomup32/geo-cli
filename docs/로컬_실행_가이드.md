# GEO CLI 로컬 실행 가이드

생성 날짜: 2026-03-26

## 개요
GEO CLI는 **Cloud(Render)**와 **로컬 PC** 간 자동 연동을 지원합니다.
- **Cloud**: 인터뷰 → 쿼리 생성 → 분석 → 보고서
- **Local**: ChatGPT 스크래핑 (Playwright)

---

## 🚀 권장: Worker 모드 (완전 자동)

Worker를 한 번 실행해 두면, 웹 앱에서 쿼리를 만들 때마다 **자동으로 감지 → 테스트 → 업로드 → 분석** 까지 진행됩니다.

### 실행 방법
```powershell
# 최초 1회: 환경 설치 + Worker 시작
.\setup_and_run.ps1 -Worker

# 이미 설치 완료된 경우
.\setup_and_run.ps1 -SkipInstall -Worker

# API Key로 인증하는 경우
.\setup_and_run.ps1 -SkipInstall -Worker -ApiKey "your-api-key"

# 직접 파이썬 명령어
python -m geo_cli.local_worker --server https://geo-cli.onrender.com
```

### 동작 흐름
1. Worker가 30초마다 클라우드 API를 확인
2. 대기 중인 작업(쿼리 완료, 테스트 미실행)이 있으면 자동 감지
3. Playwright 브라우저가 열리고 ChatGPT 스크래핑 실행
4. 완료된 결과를 클라우드에 자동 업로드
5. 클라우드에서 분석 + 보고서 자동 생성

> **참고:** 첫 실행 시 ChatGPT 로그인이 필요합니다. 한 번 로그인하면 세션이 저장됩니다.

---

## 📁 수동: 파일 다운로드 모드

웹 앱에서 쿼리 JSON을 직접 다운로드해서 실행하는 방식입니다.

```powershell
.\setup_and_run.ps1 -QueriesFile "C:\Downloads\queries_geo_xxxx.json"
```

---

## 🔧 사전 준비물
* Python 3.11 이상
* `.env` 파일에 `ANTHROPIC_API_KEY` 설정 (이미 구성됨)
* (선택) `GEO_WORKER_API_KEY` — 클라우드 API 인증용

## ❗ 트러블슈팅
| 증상 | 해결 |
|------|------|
| Playwright 오류 | `playwright install chromium` 실행 |
| ChatGPT 로그인 실패 | 브라우저에서 수동 로그인 (5분 대기) |
| 폴링 실패 | 서버 URL 확인, API Key 확인 |
| ChatGPT UI 변경 | `testing_agent.py` 상단 셀렉터 수정 |
