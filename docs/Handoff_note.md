# GEO CLI 개발 인수인계 노트 (Handoff Note)

작성일자: 2026-03-26

## 1. 최근 아키텍처 변경 배경 (Pivot to Local-First)
기존 프로젝트는 **Cloud(Render.com) 웹 서버 + 로컬 워커(Playwright 스크래핑 전용)**로 역할을 분리하여 `.json` 페이로드를 폴링 방식으로 동기화하는 복잡한 아키텍처를 가졌습니다. 이 방식은 Render의 Ephemeral Disk 초기화 문제와 FastAPI 워커 타임아웃 문제를 빈번하게 발생시켰습니다.

따라서 의논 끝에 구조를 대폭 단순화하여, 오직 사용자의 PC에서 모든 것이 실행되고 완결되는 **로컬-퍼스트(Local-First)** 아키텍처로 마이그레이션을 완료했습니다.

---

## 2. 변경된 시스템 구조 설명

### 2.1 통합된 파이프라인 (Single Streamlit App)
* **API/Worker 제거**: `api_server.py`, `local_worker.py`, `nginx.conf`, `start.sh` 등 분산 통신을 위해 존재했던 모든 리소스를 완전 삭제했습니다.
* **직접 실행 방식**: 이제 모든 파이프라인(인터뷰 → 쿼리 → 스크래핑 테스트 → 분석 → 보고서)이 `app.py` (Streamlit) 안에서 다이렉트로 전환되며 실행됩니다.
  * 특히, 기존에는 백그라운드에서 별도로 돌던 **Testing Agent (Playwright)**가 이제는 `app.py` 파이프라인 메뉴에서 "ChatGPT 테스트 실행" 버튼 클릭 시, 백그라운드 스레드를 통해 즉시 브라우저를 띄워 구동됩니다!
* **데이터 영속성 보장**: 이제 모든 생성물은 클라우드가 아닌 로컬 PC의 `data/` 경로에 즉시 저장됩니다.

### 2.2 외부 접속 (Public Access) 방법 도입
"로컬 PC를 서버처럼 쓰고, 외부망(스마트폰 등)에서는 IP나 링크로 들어오는 방식"을 채택했습니다.
* **`run_public.ps1` 스크립트 추가**: 이 스크립트를 실행하면:
  1. 먼저 Streamlit 서버가 로컬에서 조용히 실행됩니다.
  2. 그런 다음 `localtunnel` (npx)을 이용해 공유기 설정 없이도 외부에서 스마트폰으로 즉시 접속할 수 있는 **임시 퍼블릭 URL**을 화면에 출력합니다.
  3. 보안상 필요한 접속 패스워드(사용자 PC의 통신 공인 IP)도 찾아서 안내해줍니다.

## 3. 남아 있는 이슈 및 향후 업데이트 제안 (Next Steps)

1. **IP 출력 로직 보완 (`run_public.ps1`)**
   * 현재 `run_public.ps1` 스크립트 실행 환경이나 보안 설정에 따라 `$localIp`와 `$publicIp`가 화면 상단 안내 문구에 빈칸으로 출력되는 마이너 이슈가 있습니다. (단, 하단의 `localtunnel` 패스워드 부분은 동작함). 로컬 망(Wi-Fi) IP를 정확하게 뽑아주는 PowerShell 명령어의 커스텀이 필요합니다.

2. **TestingAgent 구동 환경 최적화 (`geo_cli/agents/testing_agent.py`)**
   * 웹을 통해 외부에서 스크래핑 "실행 버튼"을 누르면, 호스트(서버 본체) 역할을 하는 PC 화면에서 크롬 브라우저가 갑자기 열리는 방식입니다. 이때 다른 사용자가 PC 모니터를 만지거나 조작하면 충돌이 일어날 수 있습니다. 나중에 `headless=True` 로 전환하거나 에러 핸들링을 추가해 더 강건(Robust)하게 만들 필요가 있습니다.

3. **앱 사용자 경험 (UI/UX) 고도화**
   * 백그라운드 스레드로 Testing Agent가 실행되는 동안 Streamlit 화면 우측에 스트림 로그(`stream_log`)가 보이지만, 때때로 렌더링이 멈춰있는 것처럼 보일 수 있으니 Streamlit 진행바 컴포넌트(`st.status`) 등을 고도화하는 것을 추천합니다.
